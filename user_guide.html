Requirements

  • Minimum Java 17

Installation

  1. Intégrer le framework à votre variable d'environnement
  2. Dans le fichier web.xml contenu dans votre dossier WEB-INF, insérez la partie suivante :
    • param-value : le texte contenu dans la balise param-value sera le nom de votre package pour vos models dans le dossier class de votre projet

Utilisation

  1. Les Liens:
    ex: http://link/class-functiontocall
    • class-functiontocall:
      class : la classe en question
      functiontocall : fonction contenue dans la classe "class" qui retourne un ModelView ( classe contenue dans le framework )
    • ModelView :
      • HashMap(String,Object) mv : pour envoyer des donners dans votre vue. Pour récupérer les données, on fait appelle request.getAttribute(key) ( key : meme clé que dans l'hashmap et il suffit de caster l'objet correspondant à cette clé )
      • String view : le nom du fichier .jsp de votre vue . ex:(list.jsp)
    • Annotation des fonctions :
      • Annoter les fonctions utilisées dans les liens par la classe Url qui prend deux parametres ( String url : pour matcher l'url et la fonction ; String[] params : peut-être null, on met dedans les noms exactes des paramètres si il y en a )
  2. Envoi de données:

    On a déjà vu au-dessus comment envoyer des données par les liens, voyons comment on fait pour les formulaires.

    Pour les formulaires, l'action pointe toujours vers le lien du style class-fonction, pour les données, le nom données aux input doivent-êtres de même que celui des attributs de la classe visée.
